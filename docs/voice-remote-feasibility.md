# 어르신 음성 리모컨 — 기술 실현 방안 & MVP

또박또박의 음성인식을 활용해, **"채널 틀어줘"** 같은 말과 **큰 버튼 몇 개**로
어르신이 TV를 쉽게 쓰게 하는 리모컨. 본 문서는 *실제로 만들 수 있는 길*과 *최소 MVP 범위*를 정리한다.

---

## 1. 핵심 제약 — 순수 웹앱으론 TV 직접 제어가 어렵다
브라우저는 보안상 **로컬 네트워크 기기로의 임의 소켓 연결을 막는다.**
- 원시 TCP/UDP 소켓 불가, mDNS/SSDP 탐색 불가
- HTTPS 페이지 → 로컬 기기(ws/http)로 연결 시 **혼합 콘텐츠 + Private Network Access** 차단
- TV 페어링(프로토버프/TLS, 클라이언트 키)도 브라우저로는 구현 곤란

➡️ **결론:** TV 제어는 **네이티브 앱**(우리가 이미 가진 **Capacitor 앱**에 네이티브 네트워킹/플러그인) 또는
**가정 내 허브**(라즈베리파이 등)가 필요. 웹앱 단독은 부적합.

## 2. TV 제어 방식 비교
| 방식 | 대상 | 제어 내용 | 페어링 | 웹앱 가능? | 네이티브 난이도 |
|---|---|---|---|---|---|
| **Roku ECP** | Roku | HTTP `POST /keypress/...` | 없음 | △(CORS/PNA 제약) | ★ 매우 쉬움 |
| **LG webOS (SSAP)** | LG 스마트TV | WebSocket, 전원/음량/채널/앱실행 | 클라이언트 키(최초 TV 팝업) | ✕ | ★★ |
| **삼성 Tizen** | 삼성 스마트TV(2016+) | WebSocket, 키 이벤트(KEY_*) | 토큰(최초 TV 허용) | ✕ | ★★ |
| **Android TV Remote v2** | 구글/안드로이드TV, 일부 셋톱 | protobuf/TLS, 키·앱·음성 | 코드(화면 표시) | ✕ | ★★★ |
| **Apple TV (pyatv/Companion)** | Apple TV | 키·앱·미디어 | 코드 | ✕ | ★★★ |
| **SmartThings / LG ThinQ 클라우드** | 삼성/LG 등록기기 | 클라우드 API(OAuth) | 계정 연동 | ○(클라우드) | ★★ |
| **IR 블래스터** | 거의 모든 TV/셋톱 | 적외선 코드 | 없음 | ✕ | 폰 IR 거의 없음 → 허브 필요 |
| **HDMI-CEC** | HDMI 연결 기기 | 전원/음량 등 | 없음 | ✕ | 하드웨어 브리지 필요 |

## 3. 한국 환경 고려
- **삼성·LG 스마트TV**가 다수 → webOS/Tizen 직접 제어가 현실적 (단, 페어링 UX를 어르신용으로 단순화 필요)
- **IPTV 셋톱**(KT·SKB·LGU+)은 **독자 규격**이라 표준 프로토콜로 제어 어려움 → **IR 허브** 또는 통신사 앱 연동 필요
- 즉, "스마트TV 자체 채널/음량/앱"은 직접 제어 가능, **"IPTV 채널"은 IR/셋톱 연동**이 별도 과제

## 4. 권장 아키텍처 (MVP)
**이미 보유한 Capacitor 네이티브 앱 + 1개 TV 플랫폼 + 음성 매핑**
```
[어르신 폰: 또박또박 리모컨]
  ├─ 큰 버튼(전원/음량+−/채널+−)  ← 시각적 기본 조작
  ├─ 🎙 음성("채널 틀어줘")        ← 또박또박 음성인식 재활용
  └─ 네이티브 네트워킹
        └─(같은 와이파이)→ [스마트TV: webOS 또는 Tizen WebSocket]
```
- **왜 Capacitor:** 이미 iOS/안드로이드 앱 골격 + 음성인식 어댑터가 있음 → 여기에 TV 제어 플러그인/모듈만 추가
- **시작 플랫폼은 하나만** (가족이 쓰는 TV 브랜드). 검증 후 확장.
- 라이브러리 참고: `lgtv2`(webOS), `samsungtvws`/`ws`(Tizen), `androidtv-remote`(AndroidTV) — 네이티브/Node 사이드카로 포팅

## 5. 음성 명령 매핑 (또박또박 음성인식 재활용)
인식된 한국어 문장 → **의도(intent)** → TV 명령. 키워드/정규식 기반 단순 파서로 충분.
| 말 | 명령 |
|---|---|
| "켜줘 / 꺼줘" | 전원 on/off |
| "소리 키워줘 / 줄여줘 / 음소거" | 음량 ▲▼ / mute |
| "다음 채널 / 이전 채널 / 9번 틀어줘" | 채널 ▲▼ / 직접 입력 |
| "유튜브 틀어줘 / 넷플릭스" | 앱 실행 |
| "뉴스 틀어줘" | 지정 채널/앱(설정값) |
> 또박또박은 이미 한국어 텍스트 후처리(의문문 판별 등)를 하므로 의도 파서 추가는 작음.

## 6. MVP 단계
- **Phase 0 — 음성 명령 화면(하드웨어 0):** "채널 틀어줘"를 인식→의도 표시(실제 제어는 mock). 또박또박 안에서 바로 프로토타입 가능. *기능 검증·시연용.*
- **Phase 1 — 1개 스마트TV 실제 제어:** Capacitor 앱 + (LG 또는 삼성) WebSocket 제어 + 큰 버튼 + 음량/채널/전원. 같은 와이파이.
- **Phase 2 — 페어링 UX·확장:** 어르신용 초간단 TV 찾기/연결, 다른 브랜드 추가.
- **Phase 3(선택) — 레거시/IPTV:** IR·HDMI-CEC 허브(라즈베리파이 등)로 모든 TV 커버 → "전용 리모컨" 컨셉.

## 7. 리스크 / 메모
- 페어링·발견 UX가 어르신에겐 가장 큰 허들 → **가족이 1회 설정**하는 모델 권장
- TV 펌웨어/모델별 프로토콜 차이 큼 → 처음엔 **가족이 가진 모델 1종**으로 검증
- 수익성은 낮을 수 있으나 고령화 수요 큼 → **복지·R&D 예산 과제**(별도 기획서) 병행 가능
