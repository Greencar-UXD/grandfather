# 또박또박 — 제품 로드맵 / 아이디어

실사용자(어르신·가족) 대화에서 나온 니즈를 바탕으로 한 발전 방향 기록.

## 실사용자 피드백에서 확인된 니즈
- **화면이 작아 빨리 못 읽음** (어르신 폰이 작음) — 반복되는 핵심 불만
- **식탁 건너편 등 떨어진 사람의 말은 못 받음** — 바로 옆에 앉아야 동작
- **TV로 연결해서 보면 좋겠다** — "크니까"
- 어르신 **TV 리모컨이 너무 복잡/글씨 작음**
- (오해) 통역 서비스로 기대 → 번역 니즈도 잠재
- 무료로 풀고 **광고로 수익** 내는 방향에 긍정적

---

## ① 큰 화면(디스플레이) 모드 — ✅ 구현됨
- 말한 글자를 화면에 꽉 차는 **초대형 글씨**로 (이진 탐색 자동 크기 맞춤)
- 다크 배경 + 흰 글씨로 **원거리·TV 미러링** 대비 최대화
- 헤더의 ⛶ 버튼으로 진입, ✕ 또는 Esc로 해제, 브라우저 전체화면 연동
- 마이크 버튼 유지 → 한 기기를 TV에 미러링하고 말하기 가능
- **백엔드 불필요.** "화면이 작다"는 핵심 불만을 정면으로 해결.

## ② 2대 연동 "대화 모드" (방 공유) — 다음 작업
**문제:** 식탁 건너편 등 떨어진 사람과는 못 씀.
**해법:** 말하는 사람 기기 = 마이크, 어르신 기기/TV = 큰 글씨 화면. **방 코드**로 연결.

설계 메모:
- 실시간 동기화 백엔드 필요 (현재는 정적 호스팅이라 없음)
  - 후보 A: **Firebase Realtime DB / Firestore** 무료 티어 — 방마다 최신 텍스트 동기화 (가장 간단)
  - 후보 B: **WebRTC DataChannel (PeerJS 등)** — P2P, 시그널링 서버만 필요
- UX: "방 만들기" → 코드/QR 공유 → 상대가 입장 → 한쪽은 말하기, 다른 쪽은 큰 화면 수신
- 개인정보: 텍스트만 일시 동기화, 저장하지 않음(방 종료 시 폐기) — 개인정보처리방침 갱신 필요

## ③ 어르신 음성 리모컨 — 장기 / 별도 기획
**문제:** 어르신용 TV 리모컨이 너무 복잡하고 글씨가 작음.
**아이디어:** 음성·큰버튼 중심의 단순 리모컨.
- "채널 틀어줘", "뉴스 틀어줘", "유튜브 들어줘", "노래방 틀어줘" 등 음성 명령
- 물리/화면 버튼은 **전원·음량·채널** 정도로 최소화, 글씨/버튼 크게
- 또박또박을 **TV 화면에 연결**해 자막처럼 띄우는 것과 결합 가능

현실성:
- TV 제어가 필요 → 스마트TV 앱/HDMI-CEC/IR 등 **하드웨어·플랫폼 연동**이 핵심 (웹앱 범위를 벗어남)
- 수익성은 낮을 수 있으나 **고령 인구 증가**로 사회적 수요 큼 → 아버님 제안대로 **기획·공공예산(R&D/복지) 과제** 형태가 현실적
- 또박또박과는 별개의 venture로 분리해 추진 권장

## (보너스) 번역 모드
- 음성 → 글자 → 외국어 번역 표시. 통역 기대를 충족.
- 번역 API 필요(비용). 우선순위 낮음.
