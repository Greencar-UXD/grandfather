# tools — LG TV 제어 점검

## lg-test.js — 내 LG TV가 실제로 반응하는지 확인

앱을 빌드하기 전에, **TV와 같은 와이파이의 PC**에서 LG webOS TV가
페어링·명령에 반응하는지 빠르게 확인하는 스크립트.

### 1) 준비
- **Node 22 이상** (내장 WebSocket 사용). 구버전이면 이 폴더에서 `npm i ws`.
- PC와 TV가 **같은 와이파이**.
- **TV IP 찾기:** LG TV 리모컨 → 설정 → 네트워크 → 연결 상태 → IP 주소
  (예: `192.168.0.12`)

### 2) 실행 (대화형)
```bash
node tools/lg-test.js 192.168.0.12
```
- 처음엔 **TV 화면에 "접속 허용" 팝업**이 떠요 → 리모컨으로 **허용** 선택.
- 허용하면 `✅ 페어링 완료` → 이제 명령 입력:

```
리모컨> volup        # 소리 키우기
리모컨> voldown      # 소리 줄이기
리모컨> mute         # 음소거 (unmute 로 해제)
리모컨> chup         # 다음 채널
리모컨> chdown       # 이전 채널
리모컨> ch 7         # 7번 채널
리모컨> app 유튜브    # 유튜브 실행 (넷플릭스 도 가능)
리모컨> apps         # 설치된 앱 목록(앱 id 확인용)
리모컨> toast 안녕    # TV에 알림 띄우기 (연결 확인용으로 좋음)
리모컨> off          # 전원 끄기
리모컨> quit
```

### 3) 한 줄 실행
```bash
node tools/lg-test.js 192.168.0.12 volup
node tools/lg-test.js 192.168.0.12 toast 연결성공
```

### 결과는?
- **잘 되면** → 프로토콜이 이 TV에서 정상. 다음은 이 로직(`lg-webos.js`)을
  Capacitor 앱에 넣어 폰 리모컨으로 동작시키면 됩니다.
- **안 되면** → 증상(팝업이 안 뜸 / 연결 오류 / 특정 명령만 안 됨)을 알려주세요.
  모델별 차이(특히 IPTV 셋톱 채널)나 포트 차이를 같이 봅니다.

> 참고: 페어링 키는 `tools/.lgtv-key-<IP>.json` 에 저장되어 다음 실행부터는 팝업이 안 떠요.
> (이 키 파일은 git에 올리지 않습니다.)
